# P6.3: Brand Color Consolidation

## Problem

Brand colors are hardcoded across ~52 component files:
- `#0044CC` - Primary brand blue
- `#0033AA` - Hover state blue
- `#002B5C` - Dark blue accent
- `#FDFBF7` - Warm background
- `#1A1A1A` - Dark text

This makes theming difficult and violates DRY. If brand colors change, we'd need to update 50+ files.

## Goal

Consolidate all brand colors into CSS custom properties (design tokens) for easier maintenance and future theming.

## Tasks

- [ ] Define brand color tokens in `src/index.css`:
  ```css
  :root {
    --brand-blue: 220 100% 40%;        /* #0044CC */
    --brand-blue-hover: 220 100% 33%;  /* #0033AA */
    --brand-blue-dark: 210 100% 18%;   /* #002B5C */
    --brand-warm-bg: 45 50% 98%;       /* #FDFBF7 */
  }
  ```
- [ ] Add corresponding Tailwind config entries
- [ ] Search and replace hardcoded colors with tokens
- [ ] Test dark mode still works correctly
- [ ] Verify no visual regressions

## Acceptance Criteria

- [ ] Zero hardcoded brand hex colors in component files
- [ ] All brand colors defined in one place (index.css)
- [ ] Dark mode works correctly
- [ ] No visual changes to the app
